STEP 01 — RECEIVE

Query Ingestion & Auth

User query enters via web, mobile, or API. Auth service validates token, session is retrieved from Redis, user profile loaded.

STEP 02 — ROUTE

Orchestrator Routing

LangGraph orchestrator classifies intent (market query, news, finance, product). Routes to 1+ specialist agents with context payload.

STEP 03 — RETRIEVE

RAG + Live Data Fetch

Agent queries Pinecone vector store for relevant ET content. Live data pulled from NSE/BSE, ET APIs. Context window assembled.

STEP 04 — GENERATE

LLM Response Generation

Claude generates grounded, personalised response. Financial guardrails applied. Markdown response streamed via WebSocket.

STEP 05 — REMEMBER

Memory & Feedback Loop

Interaction stored in user memory graph. Preference signals (clicks, follow-ups) update personalisation layer for next query.

STEP 06 — MONITOR

Observability & Safety

All agent traces logged to Kafka. Latency, accuracy, and guardrail metrics reported to dashboard. PII masking enforced throughout.
